# Requisitos do Dashboard Financeiro

## Objetivo

Definir os requisitos principais para transformar o planejamento financeiro em uma aplicação web funcional.

## Requisitos funcionais

### RF01 – Exibir dados pessoais do planejamento

A aplicação deve mostrar:

- nome;
- idade considerada;
- atividade atual;
- período do planejamento.

### RF02 – Exibir cards de resumo

A aplicação deve mostrar os seguintes indicadores:

- depósito semanal;
- renda mensal;
- saldo atual;
- patrimônio projetado aos 18 anos.

### RF03 – Exibir distribuição mensal

A aplicação deve mostrar a divisão da renda mensal em categorias, com:

- nome da categoria;
- valor mensal;
- percentual;
- barra de progresso;
- ícone representativo.

### RF04 – Exibir metas financeiras

A aplicação deve mostrar metas com:

- nome da meta;
- valor projetado;
- valor alvo;
- percentual de progresso;
- barra visual.

### RF05 – Exibir gráfico de projeção

A aplicação deve exibir uma projeção acumulada de 36 meses, mostrando:

- patrimônio total;
- investimentos.

### RF06 – Permitir edição de valores

A aplicação deve permitir editar valores como:

- renda semanal;
- renda mensal;
- saldo atual;
- valor destinado a cada categoria;
- prazo do planejamento.

### RF07 – Restaurar valores padrão

A aplicação deve ter uma opção para restaurar os dados originais do planejamento.

### RF08 – Layout responsivo

A aplicação deve funcionar bem em:

- desktop;
- tablet;
- celular.

## Requisitos não funcionais

### RNF01 – Interface simples

A interface deve ser clara, com baixa carga cognitiva e fácil leitura.

### RNF02 – Visual profissional

O design deve parecer um produto real, com aparência moderna e confiável.

### RNF03 – Performance

A aplicação deve carregar rapidamente e evitar componentes desnecessariamente pesados.

### RNF04 – Acessibilidade

O dashboard deve usar bom contraste, textos legíveis e botões claros.

### RNF05 – Organização do código

Quando implementado, o projeto deve separar:

- componentes;
- páginas;
- dados;
- funções de cálculo;
- estilos.

## Requisitos futuros

- Login de usuário.
- Salvamento em banco de dados.
- Exportação em PDF.
- Envio de relatório por WhatsApp.
- Histórico mensal.
- Comparativo entre planejado e realizado.
- Modo escuro.
